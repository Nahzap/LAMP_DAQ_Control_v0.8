# Diagnóstico y Plan de Corrección: Loop Execution Engine

**Fecha:** 2026-03-25 13:26:00  
**Componente:** `DataOrientedExecutionEngine.cs`  
**Severidad:** CRÍTICA — Errores silenciosos en transición de loop

---

## 1. Diagnóstico de Problemas Encontrados

### BUG 1: Recursión infinita en loop (CRÍTICO)
- **Archivo:** `DataOrientedExecutionEngine.cs:133`
- **Problema:** `ExecuteTableAsync` se llama a sí misma recursivamente en cada loop.
- **Impacto:** Stack overflow potencial en sesiones largas. Los bloques `finally` no ejecutan hasta que TODA la cadena de recursión termina, causando leaks de timers y CancellationTokenSource.
- **Evidencia:** El `_playheadUpdateTimer` se dispone en `finally` (línea 178) pero también en línea 109. En recursión, el `finally` del frame exterior no corre hasta que el interior retorna.

### BUG 2: Fire-and-forget waveform stop tasks sin tracking (CRÍTICO)
- **Archivo:** `DataOrientedExecutionEngine.cs:281`
- **Problema:** `_ = Task.Run(async () => { ... })` lanza tareas de stop de waveform sin registrarlas.
- **Impacto:** Al hacer loop, estas tareas huérfanas siguen leyendo `_executionTimer` (un campo de instancia) que fue reemplazado por un nuevo Stopwatch. Esto causa timing corrupto y delays inexplicables.
- **Evidencia:** Delay largo al reiniciar loop — las tareas huérfanas compiten con la nueva iteración.

### BUG 3: Sin limpieza de hardware entre iteraciones de loop
- **Archivo:** `DataOrientedExecutionEngine.cs:125-134`
- **Problema:** Al reiniciar loop, no se detienen explícitamente los signal generators activos.
- **Impacto:** Canales pueden quedar generando señal residual durante la transición, causando datos incorrectos visibles en osciloscopio.

### BUG 4: IsLoopEnabled sin protección thread-safe
- **Archivo:** `DataOrientedExecutionEngine.cs:37-41`
- **Problema:** `_isLoopEnabled` es un `bool` sin `volatile` ni lock. La UI lo modifica desde el UI thread mientras el engine lo lee desde un background thread.
- **Impacto:** Race condition — el engine puede leer un valor stale del cache del CPU.
- **Evidencia en log:** `[DO EXEC ENGINE] Loop enabled: False` seguido de `[LOOP CONTROL] Loop enabled: True`.

### BUG 5: CancellationTokenSource leak en loop
- **Archivo:** `DataOrientedExecutionEngine.cs:71`
- **Problema:** En cada iteración recursiva, `_cts` se reemplaza sin disponer el anterior.
- **Impacto:** Memory leak progresivo en sesiones largas con loop habilitado.
