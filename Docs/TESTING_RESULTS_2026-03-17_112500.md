## 🔧 CORRECCIONES REALIZADAS

### Problema 1: Propiedades vs Parameters Dictionary
**Issue**: Tests usaban propiedades directas (`Voltage`, `StartVoltage`, etc.)  
**Fix**: Correcciones para usar `Parameters` dictionary correctamente
```csharp
// ❌ Antes (incorrecto)
EventType = SignalEventType.DC,
Voltage = 5.0

// ✅ Después (correcto)
EventType = SignalEventType.DC
};
evt.Parameters["voltage"] = 5.0;
```

### Problema 2: UpdateEvent no actualizaba nombre
**Issue**: Test `UpdateEvent_ExistingEvent_UpdatesCorrectly` fallaba  
**Fix**: Agregado método `UpdateNameAndColor` en SignalTable
```csharp
public void UpdateNameAndColor(int index, string newName, string newColor)
{
    Names[index] = newName;
    Colors[index] = newColor;
}
```

### Problema 3: Path incorrecto del DLL de tests
**Issue**: Runner buscaba DLL en `bin\Release\` en lugar de `bin\Release\net472\`  
**Fix**: Actualizado path en RunTests.ps1
```powershell
$testDllPath = "bin\Release\net472\LAMP_DAQ_Control_v0.8.SignalManager.Tests.dll"
```
